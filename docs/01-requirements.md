# 필지랩 요구사항 (2026-03-10 기준, v3.0 준비 중)

## 0. 진행 상태 표기 규칙
- 완료된 요구사항은 삭제하지 않고 `~~취소선~~`으로 표시한다.
- 미완료 요구사항은 일반 텍스트로 유지한다.

### 0.1 제품 방향
- 필지랩은 `단순 조회 서비스`보다 `정확도 중심 필지·구역 작업 시스템`에 가깝게 발전해야 한다.
- 제품의 핵심은 정보를 많이 보여주는 것이 아니라, 다음 작업을 더 빠르고 신뢰성 있게 수행하게 하는 것이다.
  - 필지 검토
  - 구역 구성 검토
  - 분석 결과 저장/비교/재활용
  - 파일 분석 품질 관리
- 현재 단계에서 필지랩은 상위 사업성 플랫폼 전체를 대체하려 하지 않고, 공공 토지정보 조회 서비스와 상위 사업성 분석 플랫폼 사이의 정밀 워크플로우 계층에 집중해야 한다.

### 0.2 전역 설계 원칙
- 앞으로 모든 새 기능은 `desktop / tablet / mobile` 기준으로 동시에 설계해야 한다.
- 모바일은 “보이기만 하는 수준”이 아니라, 최소한 핵심 흐름이 실제로 usable해야 한다.
- hover 전용 인터랙션, 과도한 고정폭 다중 컬럼, 모바일에서 읽기 어려운 대형 테이블은 지양한다.
- 복잡한 데스크톱 패턴에는 반드시 모바일 대응 패턴을 함께 제공해야 한다.
  - 사이드 패널 -> 바텀시트 / 드로어
  - 다중 컬럼 표 -> 요약 카드 + 펼치기 상세
  - 조밀한 툴바 -> 그룹 액션 메뉴
- 제품 메시지는 `빠른 조회`보다 `정확한 검토`, `정보 열람`보다 `업무용 워크플로우`에 맞춰야 한다.

### 0.3 이번 주 개발 초점
- 이번 주 목표는 대형 기능 확장보다 **현행 기능 안정화 + UI/UX 개선 + 작업 시스템화**다.
- 우선순위:
  1. 지도조회/구역조회 UX 안정화
  2. 페이지 전반 UI 규격 통일
  3. 파일분석을 품질 관리 도구처럼 개선
  4. 조회/저장 결과를 재사용 가능한 작업 내역으로 강화
  5. 상태/오류/신뢰도 표현과 모바일 usable 수준 정리

### 0.4 현재 구현 스냅샷
- 최신 안정 릴리즈는 `v2.2.1`이고, 현재 브랜치 작업 상태는 `v3.0 준비 중`이다.
- 이미 구현된 큰 축:
  - 개별조회 / 지도조회 / 파일조회 / 조회기록 / 인증 / 정책 페이지
  - 저장 구역 재열람 / 비교 / 리뷰 큐
  - 건축물대장 기반 구역 지표
  - 휴리스틱 AI 추천 / 이상치 / 피드백 저장
- 아직 남은 큰 과제:
  - 필지 경계 스냅/보정
  - 학습형 ML 추천 모델
  - 운영 지표/성능 증빙
  - 테스트/QA 자동화 강화

### 0.5 이번 주 마감 전략 (현재 판단)
- 이번 주는 새 기능 추가보다 **이미 만든 기능을 출시 후보 수준으로 확정하는 주간**이어야 한다.
- 따라서 이번 주 범위에서는 아래 원칙을 우선한다.
  1. 새 대형 기능 추가 금지
  2. 대규모 구조 리팩터링 지양
  3. 핵심 흐름의 UI/UX 완성도, 오류 대응, 모바일 usable 수준을 우선
  4. 정확도 향상은 새 모델 추가보다 검증/표현/튜닝 중심으로 진행
  5. 이번 주 산출물은 “기능 목록 증가”보다 “깨지지 않는 사용 흐름 + 설명 가능한 결과 + 마감 가능한 상태”여야 한다

### 0.6 이번 주 실제 우선순위에 대한 의견
- 현 시점에서 가장 위험도가 높은 영역은 다음 순서로 본다.
  1. `/map` 워크스페이스와 구역 리뷰 흐름
  2. 모바일에서의 지도 상세/드로어/테이블 usable 수준
  3. `/search` 결과 가독성 및 전 페이지 시각 규격 통일
  4. `/files`의 실패 원인/상태 이해 가능성
  5. 핵심 플로우 회귀 점검과 정확도 샘플 검증
- 즉, 이번 주에는 기능을 더 얹기보다 map/search/files/history/mypage의 현재 흐름을 정리하고, 실제 사용자가 끝까지 쓸 수 있는 상태로 만드는 것이 우선이다.

### 3.27 정확도 중심 저장 구조 (v3.x)
- 공공데이터 응답은 `raw -> normalized -> serving` 3계층으로 관리해야 한다.

- `serving` 계층은 화면/집계/다운로드에 즉시 사용하는 요약 테이블을 저장해야 한다.

### 3.33 신뢰도 및 이상치 관리 (v3.x)
- 공시지가, 세대수, 용적률, 필지 포함 판정 등 핵심 필드에는 `confidence` 개념을 도입해야 한다.
- 필드별 신뢰도는 최소 `High / Medium / Low` 또는 동등한 수치형 점수 체계로 제공해야 한다.
- 필드별 신뢰도 판정 예시는 다음을 기본값으로 사용해야 한다.
  - 공시지가: 공식 응답 직접 매칭 = High, 연도 fallback = Medium, 추정 = Low
  - 세대수: 원문 세대수 = High, 가구수 = Medium, 전유부 호수 기반 = Low
  - 용적률: 원문값 = High, 연면적/대지면적 재계산 = Medium, fallback = Low
  - 필지 포함: `overlap_ratio >= 0.95` 또는 centroid 명확 = High, score 기반 = Medium, 경계 후보 = Low
- ~~이상치 탐지는 최소 규칙 기반 탐지부터 도입해야 한다.~~
  - 미래년도 사용승인일
  - 비정상 용적률
  - 면적 0 이하
  - 동일 PNU 중복 이상
- 이후 이상치 탐지는 `Isolation Forest`, `LOF` 등 모델 기반 탐지로 확장할 수 있어야 한다.
