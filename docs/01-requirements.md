### 3.16 공간 집계(폴리곤) 조회
- 지도조회 내에 `기본 조회`와 `구역 조회`를 분리 제공해야 한다.
- 사용자는 `구역 조회`에서 폴리곤을 직접 그려 구역 분석을 요청할 수 있어야 한다.
- 구역조회 화면에서도 `지적도 보기/끄기` 토글 버튼을 제공해야 한다.
- 폴리곤 작성 UX는 최소 다음을 제공해야 한다.
  - 꼭짓점 추가/이동/삭제
  - 마지막 점 되돌리기(Undo)
  - 전체 초기화
  - 닫히지 않은 폴리곤 제출 방지
- 사용자는 분석 전 `구역 이름`을 설정해야 하며, 이름 규칙(길이/금지문자)을 검증해야 한다.
- 구역 분석 결과는 자동 저장하지 않고, 사용자가 `구역 저장` 버튼을 눌렀을 때만 저장해야 한다.
- 분석 완료 후 화면 상단에 `구역 정보 요약`을 표시해야 한다.
  - 구역명
  - 기준연도(최신연도)
  - 구역 면적(㎡)
  - 포함 필지 수
  - 평균 공시지가(총 공시지가 합계 / 구역 면적)
  - 총 공시지가 합계(최신연도 기준)
- 분석 완료 후 하단에 `필지 목록`을 표시해야 한다.
  - 각 행에는 지번주소, 도로명주소, 면적, 최신 단가(원/㎡), 면적×공시지가(원)가 포함되어야 한다.
  - 각 행 선택 체크박스를 제공하고 다중 선택 삭제(분석 결과에서 제외) 기능을 제공해야 한다.
  - 제외 즉시 요약 수치가 재계산되어야 한다.
- 지도와 목록은 연동되어야 한다.
  - 목록 행 클릭 시 해당 필지를 지도에서 강조 표시
  - 목록 주소 클릭 시 `기본 조회`로 전환하고 해당 필지의 상세 공시지가 결과를 즉시 조회
  - 지도에서 선택된 필지에 대응 행 강조
- 구역조회 결과 상단에는 `필지 포함 기준: 구역 내부 90% 이상` 안내 문구를 표시해야 한다.
- 저장된 구역 목록을 지도 우측 사이드 패널에서 즉시 열람할 수 있어야 한다.
  - 저장 구역 선택 시 폴리곤과 분석 결과를 즉시 복원
  - 저장 구역 이름 수정 가능
  - 저장 구역 삭제 가능
- 구역을 그릴 때 첫 점부터 시각적으로 표시되어야 한다.
- 각 꼭짓점에는 순서 번호를 작은 배지 형태로 표시해야 한다.

### 3.17 공간 집계 DB 확장
- `parcels.geom`(`GEOMETRY(POLYGON, 4326)`)과 GIST 인덱스를 운영에 반영해야 한다.
- 집계/반경 조회는 공간 인덱스를 활용해야 한다.
- 구역 분석 재현성과 감사 추적을 위해 다음 저장 구조를 지원해야 한다.
  - 구역 분석 마스터(요청자, 구역명, 기준연도, 임계치, 면적, 생성시각)
  - 구역-필지 상세(포함비율, 포함/제외 상태, 제외 사유/시각)
- 분석 결과는 로그아웃 이후에도 재열람 가능하도록 사용자 단위로 영속 저장해야 한다.

### 3.18 공간 집계 성능/안전 제약
- 폴리곤 최대 꼭짓점 수 100개 제한
- 폴리곤 면적 5km² 초과 요청 거부
- 집계 쿼리 타임아웃 적용
- 필지 포함 판정은 `90% 이상 포함` 규칙을 기본값으로 사용해야 한다.
  - 판정식: `ST_Area(ST_Intersection(parcel.geom, zone.geom)) / ST_Area(parcel.geom) >= 0.9`
  - 임계치(`0.9`)는 설정 가능 값으로 관리하고, 분석 메타데이터에 함께 저장해야 한다.
- 자체 교차(Self-intersection) 등 유효하지 않은 폴리곤은 서버에서 정규화/검증 후 거부 사유를 반환해야 한다.
- 동일 사용자의 동일 구역(좌표 해시 동일) 재요청은 단기 캐시로 중복 계산을 방지해야 한다.

### 3.19 공간 집계 API
- `POST /api/v1/map/zones/analyze`(가칭) 엔드포인트를 제공해야 한다.
- 입력:
  - `zone_name`
  - 폴리곤 좌표 배열
  - (선택) 포함 임계치
- 출력:
  - 구역 요약(`parcel_count`, `total_area`, `average_unit_price`, `assessed_total_price`, `base_year`)
  - 필지 목록(주소/PNU/면적/최신 단가/포함비율/상태)
  - 메타데이터(타임아웃 여부, 제외 건수, 계산시각)
- 필지 수동 제외 처리를 위한 엔드포인트를 제공해야 한다.
  - 예: `PATCH /api/v1/map/zones/{zone_id}/parcels/exclude`
- 분석 결과 다운로드를 위한 엔드포인트를 제공해야 한다.
  - 예: `GET /api/v1/map/zones/{zone_id}/export?format=csv`
