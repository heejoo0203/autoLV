# 릴리즈 노트 - v2.2.1

- 릴리즈 버전: `v2.2.1`
- 릴리즈 날짜: `2026-03-07`
- 기준 브랜치: `main`
- 이전 기준: `v2.2.0`

## 1. 릴리즈 요약
v2.2.1은 구역조회 저장 흐름과 조회기록 관리 UX를 운영 관점에 맞게 정리한 안정화 릴리즈다.

핵심 변화:
- 구역분석 결과 자동 저장 제거 -> `구역 저장` 명시 동작으로 변경
- 저장 구역 패널을 지도 우측 사이드 영역으로 이동
- 조회기록 다중 선택 삭제 추가
- 구역분석 `uuid` import 누락으로 인한 500 오류 수정
- v3.x 건축물대장 기반 노후도/용적률 분석 요구사항 및 문서 반영

## 2. 주요 기능 변경
### 2.1 구역조회 저장 워크플로우
- `/map/zones/analyze`는 이제 저장 전 미리보기만 반환
- 사용자가 `구역 저장` 버튼을 눌렀을 때만 `/map/zones`로 영속 저장
- 저장 전 결과에서도 필지 선택 제외 가능
- 저장된 결과만 CSV 다운로드 가능

### 2.2 저장 구역 UI
- 저장 구역 목록을 지도 우측 사이드 패널로 재배치
- 넓은 레이아웃으로 지도 우측 여백 활용 개선
- 저장 구역 불러오기 / 이름 수정 / 삭제 유지

### 2.3 조회기록 관리
- 체크박스 기반 다중 선택 삭제 추가
- 전체 선택 / 새로고침 / 선택 삭제 제공
- 삭제 API: `POST /api/v1/history/query-logs/delete`

## 3. 버그 수정
- 구역분석 중 `NameError: uuid is not defined` 오류 수정
  - 원인: `map_zone_service.py`의 `_upsert_parcel_geometries()`에서 `uuid.uuid4()` 사용 중 import 누락
  - 조치: `uuid` import 복구

## 4. 문서 업데이트
- 요구사항 문서에 현재 완료된 구역조회 항목 반영
- 구역분석 저장 플로우/API 명세 최신화
- 조회기록 삭제 API 추가
- v3.x 예정 기능:
  - 건축물대장 API 기반 노후도(%)
  - 평균 용적률
  - 건축지표 캐시 스키마

## 5. 다음 단계 (v3.x)
1. 건축물대장 API 연동
2. 구역별 노후도/평균 용적률 계산
3. 건축물 응답 캐시 및 호출량 절감
4. 토지이용계획 API와 결합한 상한 용적률 비교
