# 구역 분석 정확도 향상 설계안 (2026-03-10 기준)

> 현재 구현 범위는 `zone-ai-heuristic-v1.0.0` 기반 휴리스틱 추천, 이상치 탐지, 피드백 저장까지다.

## 1. 목표
구역 분석의 핵심 품질은 `필지 포함/제외 정확도`와 `실무자가 의도한 경계 재현성`이다.  
이 문서는 필지랩에 AI를 추가할 때 어떤 역할을 맡겨야 하는지, 어떤 방식이 현실적인지 정리한다.

핵심 원칙:
- 최종 계산 엔진은 계속 PostGIS + 공식 공시지가/지적도 데이터를 사용한다.
- AI는 계산을 대체하지 않고, 경계 해석/추천/설명만 담당한다.
- 사용자가 대충 그린 폴리곤의 의도를 보조하는 것이 1차 목표다.
- 사실값 / 계산값 / 추정값 / 참고값을 절대 섞지 않는다.
- 토지와 건축물은 계산 키와 집계 로직을 분리한다.
- 데이터 구조는 장기적으로 `raw -> normalized -> serving` 3계층으로 가야 한다.

현재 제품 관점:
- 필지랩의 AI는 `화려한 기능 추가`가 아니라 `정확도 중심 작업 시스템 보조`에 초점을 맞춘다.
- 현 단계에서 중요한 것은 AI 기능 수를 늘리는 것보다, 추천과 실제 최종 결정의 차이를 계속 측정하고 개선하는 검증 루프를 만드는 것이다.

## 2. 구현 상태
### 2.1 현재 구현 완료
- 휴리스틱 추천 엔진(`zone-ai-heuristic-v1.0.0`)
- `included | uncertain | excluded` 추천 상태
- `ai_confidence_score`, `ai_reason_codes`, `ai_reason_text`
- `selection_origin`, `ai_applied`
- 사용자 최종 결정 피드백 저장(`zone_ai_feedbacks`)
- 규칙 기반 이상치 탐지
- 구역 요약용 AI 설명 문장 생성
- 기능 플래그(`MAP_ZONE_AI_ENABLED`)와 임계값 설정값

### 2.2 아직 미구현
- 필지 경계 스냅/보정
- 학습형 ML 추천 모델(`LightGBM`, `XGBoost`)
- LLM 기반 장문 리포트
- raw / normalized / serving 계층 전체 도입

## 3. 우선순위
### 3.1 1순위: 필지 경계 스냅/보정
- 성격: 비AI 공간 알고리즘
- 목적: 사용자가 그린 폴리곤을 인접 필지 경계에 맞춰 정리
- 효과: 가장 직접적으로 포함 필지 정확도를 높임

예시:
- 사용자가 블록 전체를 의도했지만 손으로 비뚤게 그림
- 시스템이 인접 필지 geometry를 참고해 경계를 정렬
- 결과적으로 포함/제외 오차 감소

### 3.2 2순위: 경계 애매 필지 추천 ML
- 성격: 표 기반 머신러닝
- 목적: `90% 규칙` 경계 근처의 애매한 필지를 포함/제외/검토 필요로 추천
- 효과: 실무자가 수동으로 걸러내는 시간을 줄임

### 3.3 3순위: 설명/리포트 LLM
- 성격: LLM API
- 목적: 추천 이유 설명, 구역 요약 문장 생성
- 효과: 리포트/설명 품질 향상

비고:
- LLM은 계산값을 생성하거나 필지 포함 여부를 확정하면 안 된다.
- 딥러닝 기반 항공사진/위성영상 경계 추정은 현재 단계에서 우선순위가 낮다.

## 4. 기술 방식 비교
### 4.1 단순 LLM API
장점:
- 설명 생성이 빠름
- 리포트/사유 문장화에 유리

한계:
- geometry/수치 일관성이 낮음
- 같은 입력에도 결과가 흔들릴 수 있음
- 필지 포함/제외 계산 엔진으로는 부적합

권장 역할:
- 추천 사유 설명
- 구역 분석 요약 리포트 생성

### 4.2 전통적 머신러닝
권장 후보:
- `LightGBM`
- `XGBoost`
- `RandomForest`

장점:
- 표 형태 피처에 적합
- 학습/추론 속도가 빠름
- 설명 가능성이 높음
- 적은 데이터로도 시작 가능

권장 역할:
- 경계 애매 필지 추천 모델

### 4.3 딥러닝
장점:
- 영상/이미지 기반 해석에 강함

한계:
- 현재 데이터 구조에는 과함
- 학습/운영 비용이 큼
- 위성영상/항공사진 원본 파이프라인이 필요

권장 판단:
- 현 단계에서는 도입 보류

## 5. 추천 아키텍처
1. 사용자가 폴리곤을 그림
2. PostGIS가 기본 포함 필지 계산 (`overlap_ratio`, `overlap_area_sqm`, `centroid_in`)
3. 규칙 기반 score가 `확정 포함 / 경계 후보 / 제외`를 분류
4. 필지 경계 스냅/보정 로직이 실행됨
5. 경계 구간 필지에 대해 추천 ML 추론
6. 프론트에서 추천 포함/제외/검토 필요 표시
7. 사용자가 수동으로 확정
8. 확정 결과 저장
9. LLM이 설명/요약 리포트 생성

## 6. 추천 ML 입력 피처
필지 단위 예시 피처:
- `overlap_ratio`
- 필지 중심점의 폴리곤 내부 여부
- 필지 면적
- 폴리곤 경계까지 최소 거리
- 포함 필지와의 인접 개수
- 주변 포함 필지 비율
- 동일 블록/연속 지번 여부
- 필지의 길쭉함/compactness
- 사용자가 찍은 꼭짓점 흐름과의 거리

## 7. 추천 ML 출력
기본 출력 형식:
- `included`
- `excluded`
- `uncertain`

추가 메타데이터:
- `confidence`
- `reason_codes`
- `reason_text`
- `model_version`

## 8. 학습 데이터 수집 전략
초기에는 규칙 기반 + 사용자 피드백 저장으로 시작한다.

라벨 원천:
- 사용자가 추천 결과를 수락한 경우
- 사용자가 추천 결과를 뒤집은 경우
- 사용자가 직접 선택 삭제/복구한 경우

즉, 현재 서비스에서 발생하는 사용자 수정 행위 자체가 학습 데이터가 된다.

## 9. 운영 원칙
- AI 추천은 기본적으로 `옵션`이어야 한다.
- AI 기능이 꺼져도 기존 구역 분석은 완전하게 동작해야 한다.
- 추천 모델은 기능 플래그로 즉시 비활성화할 수 있어야 한다.
- 모델 버전, 추천 근거, 사용자 최종 결정은 모두 저장해야 한다.

## 9.1 정확도 검증 루프
- 다음 단계의 핵심은 `AI를 더 붙이는 것`보다 `정확도가 실제로 좋아지고 있는지 측정하는 것`이다.
- 최소 검증 체계:
  - 골든셋(사람이 확정한 포함/제외 사례)
  - 버전별 비교
  - 오포함 / 누락 패턴 수집
  - AI 추천과 사용자 최종 결정 차이 분석
  - 추천 채택률 / 추천 후 수정률 기록
- 운영 관점에서 이 검증 루프가 없으면, AI는 설명이 많은 기능일 뿐 정확도 향상 기능이 아니다.

## 10. 신뢰도 체계
- 공시지가
  - High: 공식 응답 직접 매칭
  - Medium: 연도 fallback
  - Low: 추정/보정
- 세대수
  - High: 원문 세대수
  - Medium: 가구수
  - Low: 전유부 호수 기반 추정
- 용적률
  - High: 원문 용적률
  - Medium: 연면적/대지면적 재계산
  - Low: fallback
- 구역 포함
  - High: `overlap_ratio >= 0.95` 또는 centroid 명확
  - Medium: 규칙 score 기반
  - Low: 경계 후보

## 11. 이상치 탐지
1차 도입은 규칙 기반으로 시작한다.
- 미래년도 사용승인일
- 비정상 용적률
- 면적 0 또는 음수
- 동일 PNU 중복 이상

이후 확장:
- `Isolation Forest`
- `Local Outlier Factor`
- rule + model ensemble

## 12. 품질 지표
- Precision
- Recall
- F1 Score
- 추천 채택률
- 추천 후 수동 수정률
- 추천으로 추가된 필지 수
- 추천으로 제외된 필지 수

## 13. 필지랩 기준 최적안
가장 현실적이고 효율적인 조합:
- 필지 경계 스냅/보정 알고리즘
- 경계 애매 필지 추천 ML (`LightGBM` 우선)
- 설명/리포트 전용 LLM

한 줄 결론:
- `정확도 향상용 핵심은 GIS + 보정 알고리즘 + 경계 추천 ML`
- `LLM은 설명 보조`
- `딥러닝은 현재 단계에서는 보류`

## 14. 이번 주 적용 원칙
- 이번 주 범위에서는 AI 확장보다 다음을 우선한다.
  1. 현재 추천/이상치 결과를 더 이해하기 쉬운 UX로 표현
  2. 경계 후보 검토 흐름 강화
  3. 저장 결과 비교/재활용 강화
  4. 모바일에서도 추천/검토 흐름이 usable하도록 정리
- 즉, AI는 지금 `추가 기능`보다 `신뢰 가능한 워크플로우 보조 계층`으로 다듬는 것이 우선이다.
