# 기능 상세 명세 (2026-03-10 기준, v3.0 준비 중)

## 0. 제품 기준
- 필지랩은 `조회 페이지 묶음`보다 `정확도 중심 필지·구역 작업 시스템`을 지향한다.
- 최신 안정 릴리즈는 `v2.2.1`이고, 현재 브랜치에는 건축물 지표와 휴리스틱 AI 보조 계층이 추가돼 있다.
- 현재 주간 개발 초점은 새 대형 기능 추가보다 다음 5가지다.
  1. 지도조회/구역조회 워크스페이스 완성도
  2. 전 페이지 UI 규격 통일
  3. 파일분석의 품질 관리 도구화
  4. 저장/이용내역의 재사용성 강화
  5. 모바일 usable 수준과 상태 표현 정리
- 앞으로 모든 새 기능은 desktop/tablet/mobile을 동시에 기준으로 설계한다.

## 1. 인증/계정
상태: 구현 완료

### 1.1 가입/로그인
- 회원가입(이메일, 비밀번호, 이름, 연락처)
- 이메일 중복 확인
- 이메일 인증코드 발송/검증
- 로그인/로그아웃
- 아이디 저장(브라우저 로컬)

### 1.2 계정 복구
- 아이디 찾기: 이름+연락처 -> 마스킹 이메일
- 비밀번호 찾기: 이메일 인증코드 기반 재설정

### 1.3 마이페이지
- 이름/연락처/프로필 이미지 수정
- 비밀번호 변경
- 서비스 약관 열람(동의 버전/시각)
- 회원 탈퇴(확인 문구 입력)
- 안드로이드 APK 다운로드 버튼

## 2. 개별조회(지번/도로명)

## 4. 파일조회
상태: 구현 완료 + 품질 진단 UX 고도화 중

### 4.1 업로드/가이드
- 표준 템플릿 다운로드
- 필수/권장 컬럼 안내
- 주소 해석 모드(`auto | jibun | road`)

### 4.2 처리 로직
- 헤더 자동 매핑 + 별칭 처리
- 주소 전처리/정규화
- 유니크 주소 키 병렬 조회
- 진행률 업데이트(5행 단위/최소 주기)

### 4.3 결과/이력
- 결과 엑셀 다운로드(완료 작업만)
- 페이지당 10개 이력 표시
- 다중 선택 삭제(처리 중 제외)
- 실패 원인 요약/가이드 표시
- 상태 필터와 작업 상태 요약 유지
- 모바일에서는 상세 업로드보다 상태 확인과 다운로드 가능 여부 확인이 우선이다.

## 5. 지도조회
상태: 구현 완료(v2) + map-first workspace 고도화 진행 중

### 5.1 입력 방식
- 지도 클릭 조회
- 주소 직접 입력 조회
- 조회기록 복원(`recordId`)

### 5.2 계산/표시
- 현재/전년 공시지가
- 증감률
- 면적
- 면적×단가
- 인근 평균(200m)
- 도로명/지번 주소 표시

### 5.3 추가 조회
- 연도별 공시지가 상세 재조회
- 토지특성 상세(지목/용도지역/용도지구)
- CSV 내보내기

### 5.4 성능
- 클릭 300ms 디바운스
- 동일 좌표 중복 호출 방지
- Redis PNU 캐시
- DB(`parcels`) 가격 캐시

### 5.5 UI/UX 원칙
- 지도는 페이지 부속 요소가 아니라 메인 캔버스다.
- 검색/툴/결과는 오버레이 또는 드로어 형태로 노출한다.
- 모바일에서는 데스크톱 사이드 패널을 바텀시트/풀 높이 드로어로 전환한다.
- 사용자가 “왜 이 필지가 선택됐는지”를 빠르게 이해할 수 있어야 한다.

## 6. 구역조회(폴리곤)
상태: 구현 완료(v2.2.1) + 정확도 고도화 2차/리뷰 워크플로우 반영

### 6.1 분석 흐름
- 지도에서 꼭짓점 선택
- `구역 분석` 실행 -> 미리보기 결과 생성
- 필요 시 필지 선택 제외
- `구역 저장` 클릭 시에만 영속 저장

### 6.2 결과 구성
- 구역명 / 기준연도 / 구역 면적 / 구역 내 필지 수
- 평균 공시지가 / 포함 필지 기준 총가치 / 구역 내부 기준 총가치
- 필지 목록(핵심표 + 상세행 확장)
- 필지 포함 기준 90% 이상 안내 문구
- 사실값/계산값/추정값 구분 안내

### 6.3 정확도 1차 고도화
- 필지별 `overlap_area_sqm`, `overlap_ratio`, `centroid_in`, `selected_by_rule`, `inclusion_mode`, `confidence_score` 저장
- 포함 상태 3단 구분
  - 확정 포함
  - 경계 후보
  - 제외
- 점수 기반 포함 판정
  - `0.6 × overlap_ratio + 0.3 × centroid_in + 0.1 × adjacency_bonus`
- 이중 총가치 제공
  - 포함 필지 기준 총가치
  - 구역 내부 기준 총가치
- CSV에 정확도 필드 포함

### 6.4 저장 구역
- 우측 사이드 패널에서 목록 확인
- 저장 구역 불러오기 / 이름 수정 / 삭제
- 저장된 구역만 CSV 다운로드 허용

### 6.5 검토 워크플로우
- 경계 후보 / AI 충돌 / 이상치 중심 리뷰 큐 제공
- 필지별 판정 근거(겹침률, 신뢰도, AI 추천, 이상치 수준, 선택 출처) 노출
- 저장 구역과 현재 분석 결과 비교 카드 제공
- 필요 시 AI 추천 포함 후보와 사용자 제외 상태를 함께 검토
- 모바일에서는 검토 큐/상세를 접고 펼치는 구조로 우선순위를 단순화

## 11. 구역 분석 정확도 향상 보조 계층 (v3.x)
상태: 2차 구현 완료 + 후속 설계 진행 중

### 11.1 기본 원칙
- 최종 구역 계산 결과(포함 필지, 면적 합, 총 공시지가)는 계속 `PostGIS + 공식 데이터` 기반 결정론적 로직으로 계산
- AI는 계산 엔진을 대체하지 않고, 경계 해석/추천/설명만 담당
- 사용자가 직접 그린 폴리곤의 실무적 의도를 더 정확히 복원하는 것이 목표
- 토지와 건축물은 내부 계산 기준 키를 분리
- 사실값 / 계산값 / 추정값을 결과 화면에서 구분

### 11.2 우선순위
- 완료:
  - `overlap_ratio`, `centroid_in`, `confidence_score` 저장
  - `확정 포함 / 경계 후보 / 제외` 구분
  - 포함 필지 기준 총가치 + 구역 내부 기준 총가치 동시 제공
  - 휴리스틱 AI 추천(`included | uncertain | excluded`)
  - AI 추천 사유/신뢰도/모델 버전 노출
  - 사용자 포함/제외 피드백 저장
  - 규칙 기반 이상치 탐지
- 1순위: 필지 경계 스냅/보정 알고리즘
- 2순위: 경계 애매 필지 추천 ML
- 3순위: 결과 설명/리포트 LLM

### 11.3 경계 스냅/보정
- 사용자가 그린 폴리곤을 인접 필지 경계에 맞춰 정렬/보정
- 대충 그린 경계를 실제 필지 묶음에 더 가깝게 보정
- 이 단계는 AI가 아니라 공간 알고리즘 우선 적용

### 11.4 경계 애매 필지 추천 ML
- 추천 대상은 `90% 포함` 기준에 걸리지 않았지만 포함 가능성이 있는 경계 필지로 제한
- 초기 구현은 딥러닝이 아니라 `LightGBM`, `XGBoost`, `RandomForest` 같은 표 기반 머신러닝 사용
- 추천 출력:
  - `included`
  - `excluded`
  - `uncertain`
- 추가 메타데이터:
  - `confidence`
  - `reason_codes`
  - `reason_text`
  - `model_version`

### 11.5 추천 피처 예시
- `overlap_ratio`
- 필지 중심점의 폴리곤 내부 여부
- 필지 면적
- 폴리곤 경계와의 최소 거리
- 포함 필지와의 인접 개수
- 주변 포함 필지 비율
- 동일 블록/연속 지번 여부

### 11.6 사용자 피드백
- AI 추천은 자동 확정하지 않고 사용자가 검토 후 최종 반영
- 사용자의 수동 제외/복구/확정 이력을 학습용 피드백 데이터로 저장
- 화면에는 `규칙 기반 포함`, `AI 추천 포함`, `AI 추천 제외`, `검토 필요`를 구분 표시
- 저장 테이블: `zone_ai_feedbacks`
- 기록 필드: `ai_model_version`, `ai_recommendation`, `final_decision`, `decision_origin`

### 11.7 현재 구현된 AI 1차 범위
- 휴리스틱 추천 모델 버전: `zone-ai-heuristic-v1.0.0`
- 기본 추천 점수는 규칙 기반 score(`confidence_score`)와 `centroid_in`, `adjacency_bonus`, 교집합 면적 보너스를 함께 사용
- 추천 출력:
  - `included`
  - `uncertain`
  - `excluded`
- 이상치 탐지:
  - 미래/비정상 사용승인년도
  - 비정상 용적률
  - 면적 0 이하
  - 가격/증감률 급격 이상
- 필드 신뢰도:
  - `building_confidence`
  - `household_confidence`
  - `floor_area_ratio_confidence`

### 11.8 LLM 역할 제한
- LLM은 필지 포함 판정 엔진으로 사용하지 않음
- 추천 이유 설명, 구역 요약 리포트, 관리자용 분석 문구 생성에 한정

### 11.9 운영 조건
- AI 기능은 끌 수 있어야 하며, 비활성화 시에도 기존 구역 분석은 그대로 동작
- 모델 버전/추천 결과/사용자 최종 결정은 추적 가능해야 함
- 품질 평가는 `Precision`, `Recall`, `F1`, 추천 채택률, 추천 후 수동 수정률 기준으로 관리
