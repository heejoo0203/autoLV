# 포트폴리오 강화 가이드 (v2.2.1)

## 1. 목적
`autoLV`를 단순 기능 구현 프로젝트가 아니라, 실제 운영형 제품 포트폴리오로 보이게 만드는 추가 작업 항목을 정리한다.

핵심 목표:
1. 성능 개선을 수치로 제시
2. 사용자 가치(Use Case)를 명확히 제시
3. 운영 지표와 장애 대응 체계를 제시

## 2. 현재 반영된 기반
- 조회 기능 3축 구축 완료: 개별조회/파일조회/지도조회
- 구역분석 기능 구축 완료: PostGIS 90% 포함 규칙, 수동 저장, 저장 구역 재열람
- 계정 기능 확장 완료: 이메일 인증, 아이디/비밀번호 찾기, 마이페이지
- 기록 기능 고도화 완료: DB 영속 저장, 필터/정렬, 페이지 복원
- 운영 안정화 기반 완료: VWorld 프록시 우회 전략(EC2 고정 IP)

## 3. 추가하면 강력한 항목
### 3.1 Redis 캐시 성능 증빙
산출물:
- README 성능 섹션
- 캡처 이미지(`docs/assets/perf-cache-compare.png`)

기록 항목:
- 측정 날짜/환경(로컬/운영)
- 입력 주소 샘플
- 1차 조회(캐시 미스) 시간
- 2차 조회(캐시 히트) 시간
- 개선 배수/개선율

README 예시 문구:
```md
### Redis 캐시 성능 비교
- 첫 조회(캐시 미스): `420ms`
- 두 번째 조회(캐시 히트): `38ms`
- 개선 배수: 약 `11x`

![Redis 캐시 성능 비교](docs/assets/perf-cache-compare.png)
```

### 3.2 실제 사용 시나리오 문서
권장 위치:
- `docs/10-user-scenarios.md` (신규)

필수 페르소나:
1. 공인중개사
2. 정비/재개발 분석 담당자
3. 토지 투자 판단 사용자

각 페르소나별 필수 항목:
- 문제 상황
- 입력 방식(개별/엑셀/지도)
- 서비스 사용 흐름
- 의사결정 결과(무엇이 빨라지고 정확해졌는지)

### 3.3 운영 지표/모니터링
권장 최소 지표:
- `api_error_rate`
- `vworld_direct_failure_rate`
- `vworld_proxy_usage_rate`

권장 도구:
- Sentry (API/Web)
- 로그 집계(CloudWatch/Datadog/Grafana/ELK 중 선택)

### 3.4 v3 차별화 포인트: 재개발 사업성 분석
추가 시 강력한 포트폴리오 포인트:
- 건축물대장 API 기반 구역 노후도 계산
- 평균 용적률/총연면적/총대지면적 집계
- 공시지가 + 건축물 지표 결합으로 “재개발 사업성 분석 도구” 포지셔닝

권장 산출물:
- 노후도/용적률 결과 카드 스크린샷
- 건축물대장 캐시 전략 문서
- 실사용 시나리오(정비사업 분석 담당자)

## 4. 면접/자기소개서 연결 포인트
- 기술 선택 근거: Next.js + FastAPI + Postgres/PostGIS + Redis
- 장애 대응 경험: VWorld 직접 호출 실패 -> 고정 IP 프록시 우회
- 성능 개선 경험: Bulk 유니크 키 병렬 조회, 진행률 미세 업데이트
- 제품 관점 경험: 비로그인/로그인 UX 분기, 정책 페이지, 앱 배포 연계
