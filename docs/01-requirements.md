### 3.11 지도 조회 (2단계 확장)
- 지도 기반 조회는 Kakao Maps JS SDK를 사용해야 한다.
- 사용자가 지도 클릭 시 아래 순서로 데이터를 계산해야 한다.
1. 클릭 좌표(lat/lng) 획득
2. VWorld API로 좌표를 PNU로 변환
3. 공공데이터 API로 당해연도 개별공시지가 조회
4. 전년도 개별공시지가 조회
5. 필지 면적 조회
6. 아래 계산값 생성
  - 현재 공시지가
  - 전년 대비 증감률(YoY)
  - `면적 × 단가`
  - 반경 200m 인근 평균가(PostGIS `ST_DWithin`)
- 계산 결과는 지도 우측(또는 하단) 결과 패널에 표시해야 한다.

### 3.12 지도 조회 성능/중복 방지 (2단계 확장)
- 지도 클릭 이벤트는 300ms 디바운스를 적용해야 한다.
- 동일 클릭/동일 PNU에 대한 중복 API 호출을 방지해야 한다.
- PNU 변환 결과는 Redis에 24시간 TTL로 캐시해야 한다.
- 공시지가 API 결과는 서버 DB(PostgreSQL)에 캐시해야 한다.

### 3.13 지도 조회 데이터 저장 (2단계 확장)
- PostgreSQL + PostGIS를 도입해야 한다.
- `parcels` 테이블을 사용해야 한다.
  - `id` (PK)
  - `pnu` (UNIQUE)
  - `lat`
  - `lng`
  - `area`
  - `price_current`
  - `price_previous`
  - `updated_at`
- 반경 조회를 위해 PostGIS `geography` 컬럼을 유지해야 한다.

### 3.14 CSV Export (2단계 확장)
- `GET /export?pnu=...` 엔드포인트를 제공해야 한다.
- CSV 컬럼은 아래 순서를 유지해야 한다.
  - `pnu`
  - `area`
  - `current_price`
  - `previous_price`
  - `growth_rate`

### 3.15 지도 조회 보안 (2단계 확장)
- 외부 API 키는 `.env`로만 관리하고 클라이언트에 노출하면 안 된다.
- 외부 API 호출은 서버(FastAPI)에서만 수행해야 한다.

### 3.16 공간 집계(폴리곤) 조회 (2단계 확장)
- 사용자는 Kakao Map에서 여러 점을 연결해 폴리곤을 그릴 수 있어야 한다.
- 프론트는 폴리곤 좌표 배열을 백엔드로 전송해야 한다.
- 백엔드는 좌표를 WKT `POLYGON((...))` 형식으로 변환해야 한다.
- 백엔드는 PostGIS `ST_Intersects` 기반 집계를 수행해야 한다.
```sql
SELECT
  COUNT(*) as parcel_count,
  SUM(price_current) as total_price,
  AVG(price_current) as avg_price,
  SUM(area) as total_area
FROM parcels
WHERE ST_Intersects(
  geom,
  ST_GeomFromText(:polygon, 4326)
);
```
- 집계 결과(`parcel_count`, `total_price`, `avg_price`, `total_area`)를 반환해야 한다.

### 3.17 공간 집계 DB 확장 (2단계 확장)
- `parcels` 테이블에 공간 컬럼을 추가해야 한다.
```sql
ALTER TABLE parcels
ADD COLUMN geom GEOMETRY(POLYGON, 4326);

CREATE INDEX idx_geom_gist
ON parcels
USING GIST (geom);
```
- 공간 집계/반경 조회는 공간 인덱스를 활용해야 한다.

### 3.18 공간 집계 성능/안전 제약 (2단계 확장)
- 폴리곤 최대 꼭짓점 수는 100개로 제한해야 한다.
- 폴리곤 면적이 5km²를 초과하면 요청을 거부해야 한다.
- 집계 쿼리에는 타임아웃을 설정해야 한다.
- 동일 요청에 대한 중복 호출 방지 로직을 적용해야 한다.

### 3.19 공간 집계 API (2단계 확장)
- `POST /aggregate` 엔드포인트를 제공해야 한다.
- 입력: 폴리곤 좌표 배열(위도/경도)
- 출력: 집계 결과와 사용된 폴리곤 메타데이터(면적, 꼭짓점 수)

### 3.20 지도 확장 구현 산출물 (2단계)
- FastAPI `/aggregate` 엔드포인트 구현
- WKT 변환 유틸리티 구현
- Kakao Map 폴리곤 드로잉/전송 예시 코드
- PostGIS 공간 집계 쿼리 예시
- Redis/DB 캐시 연계 설계 문서

## 7. 인수 기준 (2단계: 지도 조회)
- 지도 클릭 시 PNU 변환 -> 공시지가 조회 -> 계산 패널 표시가 완료되어야 한다.
- 200m 인근 평균가가 PostGIS 반경 조회로 계산되어야 한다.
- Redis 24시간 캐시가 적용되어 동일 PNU 재조회 시 응답이 빨라져야 한다.
- `GET /export?pnu=...`가 유효한 CSV 파일을 반환해야 한다.
- 폴리곤 집계 요청 시 `/aggregate`가 집계 결과를 정상 반환해야 한다.
- 꼭짓점 100개 초과/면적 5km² 초과 폴리곤은 검증 오류로 거부되어야 한다.
- 공간 집계 쿼리 타임아웃이 설정되어 장시간 락/대기 상황을 방지해야 한다.
