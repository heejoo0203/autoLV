# 릴리즈 노트 - v2.2.0

> 후속 업데이트는 `docs/10-release-notes-v2.2.1.md`를 참고한다.
> 현재 브랜치 기준 구현 상태는 `v3.0 준비 중`이며, 이 문서는 `2026-03-05` 시점 릴리즈 기록이다.

- 릴리즈 버전: `v2.2.0`
- 릴리즈 날짜: `2026-03-05`
- 기준 브랜치: `main`
- 이전 기준: `v2.1.1`

## 1. 릴리즈 요약
v2.2.0은 필지랩을 운영형 제품 수준으로 정리한 릴리즈다.

핵심 변화:
- 인증/계정 플로우 고도화(이메일 인증, 아이디/비밀번호 찾기, 약관 조회)
- 지도조회 완성도 강화(클릭+주소입력, 상세조회, CSV, 기록 연동)
- 조회기록 고급 필터/정렬
- 정책 페이지(개인정보처리방침, 계정삭제 안내) 추가
- Android Wrapper 앱 배포 경로 확정(APK/AAB)

## 2. 주요 기능 추가/개선
### 2.1 인증/계정
- 회원가입 이메일 중복 확인 API 연동
- 회원가입 이메일 인증코드 검증
- 아이디 저장(로그인 모달)
- 아이디 찾기(이름+연락처 -> 마스킹 이메일)
- 비밀번호 찾기(인증코드 기반 재설정)
- 마이페이지에서 이름/연락처/이미지 수정 지원
- 비밀번호 변경/회원 탈퇴 플로우 안정화
- 회원 약관 조회(`/auth/terms`, `/auth/terms/current`) 반영

### 2.2 지도조회
- Kakao 지도 클릭 조회 + 주소 직접 입력 조회
- 좌표->PNU 변환 및 지번/도로명 주소 동시 처리
- 면적/현재가/전년가/증감률/면적×단가/인근평균(200m) 표시
- 연도별 상세 재조회 버튼 추가
- 토지특성 상세(지목/용도지역/용도지구) 조회 추가
- CSV 내보내기 지원
- 지도조회 결과를 조회기록에 저장하고 복원 이동 지원

### 2.3 파일조회
- 업로드 UX 개선(드래그앤드롭, 가이드/템플릿 명확화)
- 처리 진행률 업데이트 단위 개선(5행 기준)
- 작업 이력 페이지네이션/다중 선택 삭제 고도화
- 완료 작업만 다운로드 버튼 노출

### 2.4 조회기록
- DB 영속 저장 기반으로 운영
- 시/도/시군구 선택형 필터
- 헤더 클릭 3단 정렬(내림차순 -> 오름차순 -> 기본)
- 정렬 컬럼: 일시/유형/주소/결과건수
- 유형별 페이지 복원(`search`/`map`)

### 2.5 UI/UX
- 상단 네비 구조 개편: 조회(드롭다운), 조회기록, 기능설명, 마이페이지
- 비로그인 기본 랜딩을 기능설명(`/features`)으로 변경
- 정책 링크를 글로벌 푸터에 통합
- 모바일 반응형 개선(v2.1 이후 보강)

### 2.6 정책/배포 대응
- 개인정보처리방침 페이지 `/privacy`
- 계정삭제 안내 페이지 `/account-deletion` (Play Console 제출 대응)
- 마이페이지에서 APK 다운로드 버튼 제공

### 2.7 Android Wrapper
- Capacitor Android Wrapper 구성
- 앱 아이콘/서명/릴리스 빌드 플로우 정리
- 2초 브랜드 스플래시 적용
- 산출물
  - APK: `apps/web/public/downloads/autoLV-android-release-v2.1.2.apk`
  - AAB: `apps/mobile/android/app/build/outputs/bundle/release/app-release.aab`

## 3. 안정화/운영
- VWorld 직접 호출 실패 시 EC2 프록시 우회 안정화 유지
- 로그아웃 응답(204) 처리 안정화 및 쿠키 삭제 옵션 일치 보강
- 도로명 파일 경로 해석 로직 개선(monorepo/단독배포 동시 대응)
- `DATABASE_URL` 정규화로 Neon/Railway URL 호환성 확보

## 4. 문서/운영 자산 정리
- 요구사항/아키텍처/API/DB/폴더구조/배포 문서 최신화
- v1 릴리즈 노트는 아카이브로 유지
- 최신 버전 릴리즈 노트(본 문서) 추가

## 5. 호환성/주의사항
- 지도조회는 로그인 사용자만 사용 가능
- Kakao JS 키의 도메인 등록 미완료 시 지도 렌더링 실패
- VWorld 네트워크 정책에 따라 프록시 fallback 구성이 필요할 수 있음

## 6. 다음 릴리즈 제안(v3)
1. 지도 폴리곤 집계(PostGIS `ST_Intersects`) 구현
2. Redis 캐시 성능 비교 결과(스크린샷) 문서화
3. 운영 지표(에러율, VWorld 실패율, 프록시 사용률) 수집/대시보드화
4. 소셜 로그인(네이버/카카오)
