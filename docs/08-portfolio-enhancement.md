# 포트폴리오 강화 가이드 (v1.x)

## 1. 목적
이 문서는 `autoLV`를 “개발 완료 프로젝트”에서 “운영 가능한 서비스 포트폴리오”로 끌어올리기 위한 강화 항목을 정의한다.

핵심 목표:
1. 성능 개선이 숫자로 보이게 만들기
2. 실제 사용자 가치가 한 페이지에서 보이게 만들기
3. 운영 지표가 실시간/주간 단위로 추적되게 만들기

## 3. 실제 사용 시나리오 페이지
## 3.1 페이지 목적
“기술적으로 무엇을 만들었는가”가 아니라 “누가 왜 사용하고 어떤 결정을 더 잘하게 되는가”를 보여준다.

## 3.2 권장 페르소나
1. 공인중개사
2. 재개발/정비 분석 담당자
3. 토지 투자 판단 사용자

## 3.3 시나리오 템플릿
### 시나리오 A: 공인중개사
- 문제: 매물 설명 시 토지 가격 근거를 즉시 제시하기 어려움
- 사용: 주소 단건 조회로 최근 연도별 공시지가 확인
- 결과: 상담 중 근거 데이터 제시, 설명 신뢰도 향상

### 시나리오 B: 정비/재개발 분석
- 문제: 후보지 다수를 수작업으로 비교하면 시간이 많이 소요됨
- 사용: 엑셀 1,000~10,000행 업로드 후 비동기 일괄 처리
- 결과: 주소별 연도 데이터 자동 정리, 비교 분석 시간 단축

### 시나리오 C: 투자 판단
- 문제: 투자 후보지의 최근 가격 추이를 빠르게 확인하기 어려움
- 사용: 조회기록과 단건 조회를 반복해 구간별 가격 추이 확인
- 결과: 감이 아닌 데이터 기반 1차 스크리닝 가능

## 4. 모니터링/운영 지표
## 4.1 최소 지표
- `api_error_rate`
- `vworld_direct_failure_rate`
- `vworld_proxy_usage_rate`

## 4.2 권장 구현
- 에러 추적: `Sentry`
- 로그 집계: API 접근 로그 + 외부 API 호출 로그
- 알림: Slack/이메일 (임계치 기반)

## 4.3 리포트 템플릿 (주간)
- 기간: `YYYY-MM-DD ~ YYYY-MM-DD`
- 총 요청수 / 에러율
- VWorld 직접 실패율
- 프록시 사용률
- 상위 장애 3건과 재발 방지 조치
